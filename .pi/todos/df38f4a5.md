Implement the `TunnelClient` class and the Cloudflare API layer for tunnel CRUD operations.

## Deliverables
- `src/api/client.ts` — low-level HTTP client wrapping `fetch` with auth headers, error handling, pagination
- `src/client.ts` — `TunnelClient` class with `.tunnels` namespace
- `src/api/types.ts` — Cloudflare API response types
- `src/tunnel.ts` — `Tunnel` object with properties (id, name, status, connections, token)
- Error types: `TunnelApiError`, `TunnelAuthError`, `TunnelNotFoundError`

## API Methods
```ts
const client = new TunnelClient({ accountId, apiToken })

// CRUD
await client.tunnels.create("name", { ingress, dns })
await client.tunnels.list({ status?, name?, search? })
for await (const t of client.tunnels.listAll()) { ... }
await client.tunnels.get("name-or-id")
await client.tunnels.delete("name", { force?, cleanupDns? })
```

## CF API Endpoints
- POST `/accounts/{id}/cfd_tunnel` — create
- GET `/accounts/{id}/cfd_tunnel` — list
- GET `/accounts/{id}/cfd_tunnel/{id}` — get
- DELETE `/accounts/{id}/cfd_tunnel/{id}` — delete
- GET `/accounts/{id}/cfd_tunnel/{id}/token` — get token
