## Goal

Update the example files in `lib/examples/` to use the new wrapper API, and add new examples showing the Effect SDK usage from `tunnel-sdk/effect`.

## Context

The `lib/examples/` directory has 7 example files showing SDK usage patterns. The wrapper API has a **breaking change**: sub-resource operations moved from `tunnel.ingress.add(...)` to `client.ingress.add(tunnelId, ...)`.

## What to update

### Wrapper API examples (update existing)

Update these files to use the new flat sub-resource API:

1. **`lib/examples/quick-expose.ts`** — Should still work as-is (expose API unchanged), verify.

2. **`lib/examples/named-tunnel.ts`** — Change:
   ```ts
   // OLD: await using connection = await tunnel.run()
   // NEW: Need to get token then use TunnelProcessService... 
   // Actually the wrapper may not expose run() directly.
   // Update to show the new API pattern.
   ```

3. **`lib/examples/streaming-logs.ts`** — The `tunnel.logs()` method no longer exists on the data object. Show the Effect approach or the wrapper equivalent.

4. **`lib/examples/multi-service.ts`** — Change `tunnel.ingress.add(...)` to `client.ingress.add(tunnel.id, ...)`, same for `tunnel.dns.ensure(...)` → `client.dns.ensure(tunnel.id, ...)`.

5. **`lib/examples/private-network.ts`** — Change `tunnel.routes.list()` to `client.routes.list(tunnel.id)`, etc.

6. **`lib/examples/preview-deploy.ts`** — Update to new API pattern.

7. **`lib/examples/config-validation.ts`** — Update to use `parseConfig` / `parseConfigFromYaml` / `parseConfigFromFile` instead of `TunnelConfig.parse` / `TunnelConfig.safeParse` / `TunnelConfig.fromFile` / `TunnelConfig.fromYaml`. Note: these are now Effects, not sync functions.

### Effect SDK examples (add new)

Add new example files showing direct Effect SDK usage:

8. **`lib/examples/effect-basic.ts`** — Create/list/delete tunnel using Effect:
   ```ts
   import { Effect } from "effect"
   import { TunnelOperations, LiveLayer } from "tunnel-sdk/effect"

   const program = Effect.gen(function*() {
     const tunnels = yield* TunnelOperations
     const tunnel = yield* tunnels.create("my-app")
     // ...
   }).pipe(Effect.provide(LiveLayer({ ... })))
   ```

9. **`lib/examples/effect-streaming.ts`** — Show event/log streaming with Effect Streams:
   ```ts
   const running = yield* process.run(token)
   yield* running.events.pipe(
     Stream.filter(e => e._tag === "Metrics"),
     Stream.runForEach(e => Effect.log(`${e.metrics.rps} req/s`)),
   )
   ```
