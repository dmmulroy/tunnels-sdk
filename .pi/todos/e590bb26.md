## Goal

Implement the `TunnelProcessService` (running cloudflared with full event streaming) and the `expose` convenience function (anonymous quick tunnels). These are the most complex pieces — they replace EventEmitter-based process management with Effect Streams, SubscriptionRef, and ChildProcessSpawner.

## Context

**Current `TunnelProcess`** (`packages/tunnel-sdk/src/process.ts`, ~250 lines): Spawns cloudflared via `node:child_process`, parses stderr JSON lines via readline, tracks connectors in a Map, emits events via EventEmitter, manages SIGTERM/SIGKILL shutdown.

**Current `expose`** (`packages/tunnel-sdk/src/expose.ts`, ~80 lines): Spawns cloudflared with `--url` flag, waits for trycloudflare URL in stderr, returns `{ url, close, [Symbol.asyncDispose] }`.

**Effect replacements**:
- `ChildProcess.make` + `ChildProcessSpawner.spawn` replaces `node:child_process.spawn`
- `handle.stderr` (already `Stream<Uint8Array>`) replaces readline
- `SubscriptionRef<TunnelStatus>` replaces EventEmitter status tracking
- `Scope` auto-kills process on close (replaces manual SIGTERM/SIGKILL)
- `Stream<TunnelEvent>` replaces EventEmitter events

**Layer depends on**: `ChildProcessSpawner.ChildProcessSpawner`, `CloudflaredBinary`

**Implementation** (see EFFECT_REFACTOR_PLAN.md "TunnelProcess" section):

1. `yield* binary.ensureInstalled()` to get binary path
2. Build command: `ChildProcess.make(binaryPath, ["tunnel", "--no-autoupdate", "run", "--token", token, ...options])`
3. `yield* spawner.spawn(command)` — scoped, auto-killed on scope close
4. Create `SubscriptionRef.make<TunnelStatus>("inactive")` and `SubscriptionRef.make<ReadonlyArray<ConnectorInfo>>([])`
5. Parse `handle.stderr` through `Stream.decodeText() |> Stream.splitLines`
6. For each line, try JSON parse. The parsing logic is in the current `handleLogEntry` and `tryParseUnstructuredLine` methods — port this logic:
   - Detect connected events (registered, connectionRegistered, "Registered tunnel connection")
   - Detect disconnected events (unregistered, disconnect)
   - Detect reconnecting events
   - Detect error/fatal level entries
   - Parse metrics entries
   - Update `statusRef` based on connector count (>=4 = healthy, >0 = degraded, 0 = inactive)
   - Update `connectorsRef` on connect/disconnect
7. The `events` stream should be constructed from the parsed stderr lines
8. `logs` stream: same stderr lines parsed into `LogEntry` format (reuse logic from `packages/tunnel-sdk/src/logs.ts`)
9. `waitUntilHealthy`: `SubscriptionRef.changes(statusRef) |> filter("healthy") |> take(1) |> runDrain |> timeoutFail(60s)`
10. `exitCode`: `handle.exitCode`

**Important**: The stderr stream needs to be consumed (forked) so that events flow even if the consumer doesn't read `events`. Consider using `Stream.broadcast` or forking a fiber that pipes stderr into a PubSub that both `events` and `logs` read from. Alternatively, use `Stream.share` to allow multiple consumers of the same stream.

### File: `packages/tunnel-sdk/src/effect/expose.ts`

```ts
export const expose = Effect.fn("expose")(
  function*(port: number): Effect.fn.Return<{ readonly url: string }, TunnelProcessError | BinaryInstallError, Scope> {
    const spawner = yield* ChildProcessSpawner.ChildProcessSpawner
    const binary = yield* CloudflaredBinary
    const binaryPath = yield* binary.ensureInstalled()

    const command = ChildProcess.make(binaryPath, ["tunnel", "--url", `http://localhost:${port}`])
    const handle = yield* spawner.spawn(command).pipe(
      Effect.mapError((cause) => new TunnelProcessError({ message: "Failed to spawn", cause }))
    )

    const url = yield* handle.stderr.pipe(
      Stream.decodeText(),
      Stream.splitLines,
      Stream.filterMap((line) => {
        const match = line.match(/(https:\/\/[a-zA-Z0-9-]+\.trycloudflare\.com)/)
        return match ? Option.some(match[1]) : Option.none()
      }),
      Stream.take(1),
      Stream.runHead,
      Effect.flatten,
      Effect.timeoutFail({
        duration: "30 seconds",
        onTimeout: () => new TunnelProcessError({ message: "Timed out waiting for tunnel URL" }),
      }),
    )

    return { url }
  }
)
```
