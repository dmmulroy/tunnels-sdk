## Goal

Port the Zod-based `TunnelConfig` validation to Effect Schema. This replaces `packages/tunnel-sdk/src/config/schema.ts`.

## Context

The current config module (`packages/tunnel-sdk/src/config/schema.ts`) uses Zod schemas to validate tunnel configuration (ingress rules, routes, warp-routing, origin request settings). It provides `TunnelConfig.parse()`, `TunnelConfig.safeParse()`, `TunnelConfig.fromFile()`, and `TunnelConfig.fromYaml()`.

Key validation logic to preserve:
1. **Duration strings** — regex pattern `^(\d+(\.\d+)?)\s*(ns|us|µs|ms|s|m|h)$`
2. **Service URLs** — must start with `http://`, `https://`, `tcp://`, `ssh://`, `rdp://`, `unix:`, or `http_status:`
3. **Hostnames** — regex for valid hostname or wildcard
4. **CIDR notation** — regex for route networks
5. **Origin request config** — strict object (rejects unknown keys) with duration, boolean, number, string fields
6. **Ingress rules** — array, min 1 rule, duplicate hostname detection, catch-all rule validation
7. **Auto-fallback** — if `autoFallback` is true (default) and last rule has a hostname, auto-append `{ service: "http_status:404" }`
8. **YAML parsing** — parse YAML strings and files into validated config

## What to create

### File: `packages/tunnel-sdk/src/effect/config.ts`

Port all Zod schemas to Effect Schema equivalents:

```ts
import { Effect, Schema } from "effect"
import { parse as parseYaml } from "yaml"
import { ConfigValidationError } from "./errors.js"

// Schema refinements for duration strings, service URLs, hostnames, CIDRs
const DurationString = Schema.String.pipe(Schema.pattern(/^(\d+(\.\d+)?)\s*(ns|us|µs|ms|s|m|h)$/))
// ... etc

// OriginRequest, IngressRuleConfig, TunnelConfigInput as Schema.Class

// Effectful parse with custom validations (duplicate hostnames, catch-all):
export const parseConfig = Effect.fn("TunnelConfig.parse")(function*(input) { ... })
export const parseConfigFromYaml = Effect.fn("TunnelConfig.fromYaml")(function*(yaml: string) { ... })
export const parseConfigFromFile = Effect.fn("TunnelConfig.fromFile")(function*(path: string) { ... })
```

The `parseConfig` function should:
1. Decode input through the Schema
2. Check for duplicate hostnames (custom validation, same logic as current `superRefine`)
3. Check that last rule is catch-all or `autoFallback` is true
4. Auto-append catch-all if needed
5. Return `ConfigValidationError` on failure
