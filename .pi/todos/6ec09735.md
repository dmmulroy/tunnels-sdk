## Goal

Implement the four resource manager services. These are the simplest services â€” each depends only on `CloudflareApi` and translates domain operations into API calls. VNetManager is the simplest and should be done first as proof-of-concept.

## Context

Currently these are classes that take an `IApiClient` (and sometimes a `tunnelId`) in their constructor. In Effect, they become `ServiceMap.Service` definitions with methods that take `tunnelId` as a parameter (since services are singletons, not per-tunnel instances).

## What to create

### 1. `packages/tunnel-sdk/src/effect/services/VNetManager.ts` (do first â€” simplest)

Current: `packages/tunnel-sdk/src/managers/vnets/vnet-manager.ts`

```ts
export class VNetManager extends ServiceMap.Service<VNetManager, {
  create(name: string, options?: { default?: boolean; comment?: string }): Effect.Effect<VNet, TunnelApiError | TunnelAuthError>
  delete(name: string): Effect.Effect<void, TunnelApiError | TunnelAuthError | TunnelSdkError>
  list(): Effect.Effect<ReadonlyArray<VNet>, TunnelApiError | TunnelAuthError>
}>()(
  "tunnel-sdk/VNetManager"
)
```

Layer depends on `CloudflareApi`. Implementation:
- `create`: POST to `/teamnet/virtual_networks`, map `CfVirtualNetwork` â†’ `VNet`
- `delete`: list vnets, find by name, DELETE by ID. Fail with `TunnelSdkError` if not found.
- `list`: GET `/teamnet/virtual_networks`, map results

### 2. `packages/tunnel-sdk/src/effect/services/DnsManager.ts`

Current: `packages/tunnel-sdk/src/managers/dns/dns-manager.ts`

**Important**: This service uses `Cache` from `effect/Cache` for zone ID lookups (replacing the bare `Map<string, string>` in the current code). See EFFECT_REFACTOR_PLAN.md "Cache" section.

```ts
export class DnsManager extends ServiceMap.Service<DnsManager, {
  ensure(tunnelId: string, hostname: string, options?: { proxied?: boolean; ttl?: number }): Effect.Effect<void, TunnelApiError | TunnelAuthError | TunnelSdkError>
  remove(tunnelId: string, hostname: string): Effect.Effect<void, TunnelApiError | TunnelAuthError>
  list(tunnelId: string): Effect.Effect<ReadonlyArray<DnsRecord>, TunnelApiError | TunnelAuthError>
}>()(
  "tunnel-sdk/DnsManager"
)
```

In the layer constructor:
```ts
const zoneCache = yield* Cache.make<string, string, TunnelApiError | TunnelAuthError>({
  capacity: 100,
  timeToLive: Duration.minutes(10),
  lookup: (hostname) => findZoneId(api, hostname),
})
```

Zone lookup logic (from current `findZoneId`): walk hostname parts from left to right, querying `/zones?name=<part>` until a match is found.

CNAME target: `${tunnelId}.cfargotunnel.com`

- `ensure`: find zone â†’ find existing record â†’ create or update CNAME
- `remove`: find zone â†’ find existing record â†’ delete
- `list`: paginate all zones, for each zone query DNS records with `content: cnameTarget`

### 3. `packages/tunnel-sdk/src/effect/services/IngressManager.ts`

Current: `packages/tunnel-sdk/src/managers/ingress/ingress-manager.ts`

```ts
export class IngressManager extends ServiceMap.Service<IngressManager, {
  list(tunnelId: string): Effect.Effect<ReadonlyArray<IngressRule>, TunnelApiError | TunnelAuthError>
  add(tunnelId: string, rule: IngressRule): Effect.Effect<void, TunnelApiError | TunnelAuthError | TunnelSdkError>
  remove(tunnelId: string, hostname: string): Effect.Effect<void, TunnelApiError | TunnelAuthError | TunnelSdkError>
  set(tunnelId: string, rules: ReadonlyArray<IngressRule>): Effect.Effect<void, TunnelApiError | TunnelAuthError>
}>()(
  "tunnel-sdk/IngressManager"
)
```

Key behavior to preserve:
- `add` inserts before the catch-all (last rule without hostname)
- `add` rejects duplicate hostnames
- `set` auto-appends catch-all `{ service: "http_status:404" }` if last rule has a hostname
- `set` reads current config first (to preserve `warp-routing`, `originRequest`, etc.)

### Tests

For each service, create a test file: `packages/tunnel-sdk/src/effect/services/{Service}.test.ts`

**Test strategy**: Provide a mock `CloudflareApi` layer that returns canned data. Do NOT mock HttpClient here â€” mock at the CloudflareApi boundary.

```ts

const TestCloudflareApi = Layer.succeed(CloudflareApi, CloudflareApi.of({
  get: Effect.fn(function*<T>(path: string, params?: Record<string, string>) {
    // return canned responses based on path
  }),
  post: ..., put: ..., delete: ...,
  paginate: (path) => Stream.fromIterable([...]),
  accountPath: (path) => `/accounts/test-account${path}`,
  zonePath: (zoneId, path) => `/zones/${zoneId}${path}`,
}))
```

Key behaviors per service:

**VNetManager**: create returns VNet, delete finds by name then deletes, list maps results
**DnsManager**: ensure creates CNAME, ensure updates existing record, remove deletes, zone cache prevents repeated lookups
**IngressManager**: add inserts before catch-all, add rejects duplicates, set auto-appends catch-all
**RouteManager**: add with vnet resolves name, check returns null on 404, remove fails if not found
