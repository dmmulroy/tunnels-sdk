## Goal

Create the foundational error types and domain schemas for the Effect SDK. These are pure data definitions with zero service dependencies — everything else in the Effect SDK builds on top of them.

## Context

We are refactoring `packages/tunnel-sdk` from class-based async/await to Effect TS. Full design is in `EFFECT_REFACTOR_PLAN.md`.

**Current errors** (`packages/tunnel-sdk/src/errors.ts`): Plain class hierarchy — `TunnelSdkError`, `TunnelApiError`, `TunnelAuthError`, `TunnelNotFoundError`. These extend `Error` and use manual `this.name = "..."`.

**Current types** (`packages/tunnel-sdk/src/api/types.ts`): Plain TypeScript interfaces for Cloudflare API wire types (`CfTunnel`, `CfTunnelConnection`, `CfDnsRecord`, etc.) and SDK domain types are scattered across manager files.

**Effect approach**: Use `Schema.TaggedErrorClass` for errors (gives `_tag` discriminant for `Effect.catchTag`) and `Schema.Class` for domain types (gives runtime validation + type inference).

## What to create

### File: `packages/tunnel-sdk/src/effect/errors.ts`

Define these tagged error classes using `Schema.TaggedErrorClass`:

```ts
import { Schema } from "effect"

export class TunnelSdkError extends Schema.TaggedErrorClass<TunnelSdkError>()(
  "TunnelSdkError",
  { message: Schema.String, cause: Schema.optional(Schema.Defect) }
) {}

export class TunnelApiError extends Schema.TaggedErrorClass<TunnelApiError>()(
  "TunnelApiError",
  {
    status: Schema.Number,
    errors: Schema.Array(Schema.Struct({ code: Schema.Number, message: Schema.String })),
  }
) {}

export class TunnelAuthError extends Schema.TaggedErrorClass<TunnelAuthError>()(
  "TunnelAuthError",
  {
    message: Schema.optionalWith(Schema.String, {
      default: () => "Authentication failed. Check your API token and account ID.",
    }),
  }
) {}

export class TunnelNotFoundError extends Schema.TaggedErrorClass<TunnelNotFoundError>()(
  "TunnelNotFoundError",
  { tunnelRef: Schema.String }
) {}

export class TunnelProcessError extends Schema.TaggedErrorClass<TunnelProcessError>()(
  "TunnelProcessError",
  { message: Schema.String, exitCode: Schema.optional(Schema.Number), cause: Schema.optional(Schema.Defect) }
) {}

export class BinaryInstallError extends Schema.TaggedErrorClass<BinaryInstallError>()(
  "BinaryInstallError",
  { message: Schema.String, cause: Schema.optional(Schema.Defect) }
) {}

export class ConfigValidationError extends Schema.TaggedErrorClass<ConfigValidationError>()(
  "ConfigValidationError",
  {
    message: Schema.String,
    issues: Schema.Array(Schema.Struct({
      path: Schema.Array(Schema.Union(Schema.String, Schema.Number)),
      message: Schema.String,
    })),
  }
) {}
```

### File: `packages/tunnel-sdk/src/effect/schemas.ts`

Define domain types as `Schema.Class`. Two categories:

**Cloudflare API wire types** (match the JSON the API returns):
- `CfTunnelConnection` — `{ id, colo_name, origin_ip, opened_at, client_version, is_pending_reconnect }`
- `CfTunnel` — `{ id, name, status, created_at, deleted_at, remote_config, connections }`
- `CfDnsRecord` — `{ id, name, type, content, proxied, ttl }`
- `CfZone` — `{ id, name, status }`
- `CfRoute` — `{ id, network, tunnel_id, tunnel_name?, virtual_network_id?, comment?, created_at, deleted_at }`

- `CfIngressRule` — `{ hostname?, service, path?, originRequest? }`
- `CfTunnelConfig` — `{ config: { ingress, warp-routing?, originRequest? } }`

- `Route` — `{ network, tunnelId, tunnelName?, vnet, comment? }`
- `RouteCheckResult` — `{ tunnel, route, vnet }`
- `DnsRecord` — `{ hostname, type, content }`

- `ConnectorInfo` — `{ id, colo, ip, location }`

## Effect patterns reference

```ts
// Schema.TaggedErrorClass — from .dependencies/effect-smol/LLMS.md
import { Schema } from "effect"
export class MyError extends Schema.TaggedErrorClass<MyError>()("MyError", {
  message: Schema.String
}) {}

// Schema.Class — for domain types
export class MyType extends Schema.Class<MyType>("MyType")({
  id: Schema.String,
  name: Schema.String,
  count: Schema.optional(Schema.Number),
}) {}

// @effect/vitest testing
import { assert, describe, it } from "@effect/vitest"
import { Effect } from "effect"

it.effect("catches tagged error", () => Effect.gen(function*() {
  const result = yield* Effect.fail(new MyError({ message: "boom" })).pipe(
    Effect.catchTag("MyError", (e) => Effect.succeed(e.message))
  )
  assert.strictEqual(result, "boom")
}))
```
