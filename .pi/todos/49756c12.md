## Goal

Set up the `tunnel-sdk` package for the Effect refactor by adding dependencies and configuring the new `tunnel-sdk/effect` entry point. This is pure build/config work — no source code changes.

## Context

We are refactoring `packages/tunnel-sdk` from plain async/await TypeScript to Effect TS. The full design is in `EFFECT_REFACTOR_PLAN.md` at the repo root.

The SDK will have two entry points:
- `tunnel-sdk` — thin async/await wrapper (existing, will be rewritten later)
- `tunnel-sdk/effect` — pure Effect SDK (new)

## What to do

### 1. Update `packages/tunnel-sdk/package.json`

Add these dependencies:
```json
{
  "dependencies": {
    "effect": "beta",
    "@effect/platform-node": "beta",
    "yaml": "^2.7.1"
  },
  "devDependencies": {
    "@effect/vitest": "beta",
    "@types/node": "^25.5.0",
    "tsdown": "^0.21.7",
    "typescript": "^5.8.2",
    "vitest": "^3.1.1"
  }
}
```

Remove `zod` from dependencies (it will be replaced by Effect Schema).

### 2. Add the `./effect` export to `package.json`

Add a new export map entry alongside the existing `.` and `./bin`:
```json
{
  "exports": {
    ".": { ... },
    "./effect": {
      "types": {
        "import": "./dist/effect/index.d.mts",
        "require": "./dist/effect/index.d.cts"
      },
      "import": "./dist/effect/index.mjs",
      "require": "./dist/effect/index.cjs"
    },
    "./bin": { ... }
  }
}
```

### 3. Update tsdown config

Read `packages/tunnel-sdk/tsdown.config.ts` and ensure it builds the `src/effect/index.ts` entry point in addition to the existing `src/index.ts` and `src/bin/index.ts`.

### 4. Create the directory structure

Create empty placeholder files so the build doesn't break:
- `packages/tunnel-sdk/src/effect/index.ts` — just `export {}` for now

### 5. Run install and verify

```bash
cd /Users/dillon/Code/personal/cf-tunnels
pnpm install
cd packages/tunnel-sdk
pnpm typecheck
```

## Acceptance criteria

- [ ] `pnpm install` succeeds with new deps
- [ ] `zod` is removed from dependencies
- [ ] `effect`, `@effect/platform-node`, `@effect/vitest` are installed
- [ ] `package.json` exports include `./effect`
- [ ] `pnpm typecheck` passes
- [ ] The build config includes the new entry point
