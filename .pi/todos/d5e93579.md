Implement running named tunnels via cloudflared with typed event emitter and connection management.

## Deliverables
- `src/process.ts` — `TunnelProcess` class
  - Spawns `cloudflared tunnel run --token <token>` 
  - Parses cloudflared structured log output (JSON lines from stderr)
  - Maps to typed events: connected, disconnected, reconnecting, error, metrics, status, exit
- `src/events.ts` — Typed event emitter (no `any`)
  - `TunnelProcessEvents` interface
  - `ConnectorInfo`, `ReconnectAttempt`, `TunnelError`, `TunnelMetrics` types
- Connection management:
  - `waitUntilHealthy()` — resolves when 4 connections established
  - `waitUntilExit()` — resolves when process exits
  - `.status` getter (healthy/degraded/inactive/down)
  - `.connectors` array
  - `.close()` — graceful shutdown (SIGTERM + grace period)
  - `[Symbol.asyncDispose]()` — for `using`
- `AbortSignal` support via `RunOptions.signal`

## Typed Events
```ts
connection.on("connected", (conn: ConnectorInfo) => { ... })
connection.on("disconnected", (conn: ConnectorInfo) => { ... })
connection.on("reconnecting", (attempt: ReconnectAttempt) => { ... })
connection.on("error", (err: TunnelError) => { ... })
connection.on("metrics", (m: TunnelMetrics) => { ... })
connection.on("status", (s: TunnelStatus) => { ... })
```
