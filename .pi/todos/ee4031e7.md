## Goal

Implement the `TunnelOperations` service — tunnel CRUD operations (create, list, get, delete, getToken, refresh). This is the orchestrator that uses `CloudflareApi` directly and optionally coordinates with `IngressManager`, `DnsManager`, and `RouteManager` during `create`.

## Context

Current implementation: `packages/tunnel-sdk/src/tunnel-operations.ts` (the `TunnelOperations` class) + `packages/tunnel-sdk/src/tunnel.ts` (the `Tunnel` class with `refresh`, `getToken`, `delete`, `run`, `logs`).

**Key design change**: The old SDK returned mutable `Tunnel` class instances that held data + behavior + sub-managers. The Effect version returns plain `TunnelInfo` data objects (from `schemas.ts`). Operations like `refresh`, `getToken`, `delete` live on the `TunnelOperations` service and take a tunnel ID.

## What to create

### File: `packages/tunnel-sdk/src/effect/services/TunnelOperations.ts`

```ts
export interface CreateTunnelOptions {
  readonly ingress?: ReadonlyArray<IngressRule>
  readonly dns?: { auto?: boolean; cleanup?: boolean }
  readonly routes?: ReadonlyArray<{ network: string; vnet?: string; comment?: string }>
}

export interface TunnelListOptions {
  readonly status?: TunnelStatus
  readonly name?: string
  readonly search?: string
}

export interface DeleteOptions {
  readonly force?: boolean
  readonly cleanupDns?: boolean
}

export class TunnelOperations extends ServiceMap.Service<TunnelOperations, {
  create(name: string, options?: CreateTunnelOptions): Effect.Effect<TunnelInfo, TunnelApiError | TunnelAuthError>
  list(options?: TunnelListOptions): Effect.Effect<ReadonlyArray<TunnelInfo>, TunnelApiError | TunnelAuthError | TunnelSdkError>
  listAll(): Stream.Stream<TunnelInfo, TunnelApiError | TunnelAuthError>
  get(nameOrId: string): Effect.Effect<TunnelInfo, TunnelApiError | TunnelAuthError | TunnelNotFoundError>
  delete(nameOrId: string, options?: DeleteOptions): Effect.Effect<void, TunnelApiError | TunnelAuthError | TunnelNotFoundError>
  getToken(tunnelId: string): Effect.Effect<string, TunnelApiError | TunnelAuthError>
  refresh(tunnelId: string): Effect.Effect<TunnelInfo, TunnelApiError | TunnelAuthError>
}>()(
  "tunnel-sdk/TunnelOperations"
)
```

**Layer depends on**: `CloudflareApi`, `IngressManager`, `DnsManager`, `RouteManager`

**Implementation details** (port from current `tunnel-operations.ts` and `tunnel.ts`):

1. **`create`**: POST to `/cfd_tunnel` with `{ name, tunnel_secret: generateSecret(), config_src: "cloudflare" }`. Map `CfTunnel` → `TunnelInfo`. Then if options provided:
   - If `options.ingress` → call `IngressManager.set(tunnelId, rules)`
   - If `options.dns.auto` and ingress has hostnames → call `DnsManager.ensure` for each
   - If `options.routes` → call `RouteManager.add` for each

2. **`list`**: GET `/cfd_tunnel` with params `{ is_deleted: "false", status?, name? }`. Fail with `TunnelSdkError` if both `name` and `search` are provided.

3. **`listAll`**: Use `api.paginate<CfTunnel>("/cfd_tunnel", { is_deleted: "false" })`, map each to `TunnelInfo`

4. **`get`**: If UUID format → GET `/cfd_tunnel/${id}`. Otherwise → `list({ name })` then find exact match or throw `TunnelNotFoundError`.

5. **`delete`**: `get(nameOrId)` first. If `cleanupDns` → list DNS records via `DnsManager.list`, remove each. Then DELETE `/cfd_tunnel/${id}` with `?cascade=true` if force.

6. **`getToken`**: GET `/cfd_tunnel/${id}/token`

7. **`refresh`**: GET `/cfd_tunnel/${id}`, map to `TunnelInfo`

**Helper**: `generateSecret()` — `crypto.getRandomValues(new Uint8Array(32))` → base64. Can use `Effect.sync`.

**UUID check**: `/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i`

**CfTunnel → TunnelInfo mapping**: Map `status` string through the valid set `["healthy", "inactive", "degraded", "down"]` defaulting to "inactive". Map `connections` array. Parse date strings.
