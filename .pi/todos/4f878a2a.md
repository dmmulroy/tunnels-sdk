## Location
`packages/tunnel-sdk/src/effect/services/TunnelProcess.ts` line 107

## Current state
The `LiveTunnelProcess.run()` method spawns `cloudflared tunnel run --token <token>` and returns a `RunningTunnel`. Currently the `events` and `logs` streams are stubs:
```ts
// TODO: Parse stderr for events, connectors, metrics
const events: Stream.Stream<TunnelEvent> = Stream.empty
const logs: Stream.Stream<LogEntry> = Stream.empty
```

## What to implement
cloudflared writes structured JSON logs to **stderr**. Parse these into two live streams:

### 1. `logs: Stream.Stream<LogEntry>`
Each stderr line is a JSON object like:
```json
{"level":"info","time":"2024-01-15T10:30:00Z","event":"tunnelConnection","message":"Registered tunnel connection","connectorID":"abc123"}
```
Parse into `LogEntry` schema (defined in `schemas.ts`):
- `timestamp: Date` — from `time` field
- `level: LogLevel` — from `level` field  
- `event: string` — from `event` field
- `message: string` — from `message` field
- `connectorId?: string` — from `connectorID` field

### 2. `events: Stream.Stream<TunnelEvent>`
Derive structured events from parsed log entries based on `event` field:
- `"tunnelConnection"` → `{ _tag: "Connected", connector: ConnectorInfo }`
- `"tunnelDisconnect"` → `{ _tag: "Disconnected", connector: ConnectorInfo }`
- `"reconnect"` → `{ _tag: "Reconnecting", attempt: ReconnectAttempt }`
- `"error"` → `{ _tag: "Error", error: TunnelProcessErrorInfo }`
- `"metrics"` → `{ _tag: "Metrics", metrics: TunnelMetrics }`
- Status transitions → `{ _tag: "Status", status: TunnelStatus }`

Also update `statusRef` and `connectorsRef` as events arrive (e.g., on Connected, add to connectors and set status to "healthy").

### Implementation approach
1. Create a `Stream` from `proc.stderr` using `Stream.fromAsyncIterable` or by pushing lines through a `Queue`
2. Parse each line as JSON, decode with `Schema.decodeUnknownEffect(LogEntry)`, skip unparseable lines
3. Broadcast the parsed log stream (so both `logs` and `events` can consume it)
4. Map the log stream to `TunnelEvent` based on `event` field, filtering out non-event log lines

### Types involved (from `schemas.ts` and `TunnelProcess.ts`)
- `LogEntry` — Schema.Class with timestamp, level, event, message, connectorId?
- `ConnectorInfo` — Schema.Class with id, colo, ip, location
- `TunnelMetrics` — Schema.Class with rps, p50Ms, p99Ms, activeConns, bytesIn, bytesOut
- `TunnelEvent` — tagged union (Connected | Disconnected | Reconnecting | Error | Metrics | Status)
- `TunnelProcessErrorInfo` — { code, message, retryable, connector? }
- `ReconnectAttempt` — { number, delay, connector }
