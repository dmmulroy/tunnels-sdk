Implement Zod-powered config validation with actionable error messages.

## Deliverables
- `src/config/schema.ts` ‚Äî Zod schemas for `TunnelConfig`, `IngressRule`, `OriginRequestConfig`
- `src/config/validate.ts` ‚Äî `TunnelConfig.parse()`, `.safeParse()`, `.fromFile()`, `.fromYaml()`
- Validation rules:
  - Ingress must have ‚â•1 rule
  - Last rule must be catch-all (no hostname) ‚Äî or `autoFallback: true` auto-appends one
  - Hostnames must be valid domains
  - Service URLs validated (scheme: http, https, tcp, ssh, rdp, http_status)
  - No duplicate hostnames
  - Unknown keys are errors (`.strict()`) with did-you-mean suggestions
  - `originRequest` fields validated (duration strings, valid ranges)
- YAML parsing with `yaml` package
- Actionable error messages (not just "invalid")

## Exported API
```ts
import { TunnelConfig } from "tunnel-sdk"
TunnelConfig.parse({ ... })        // throws on invalid
TunnelConfig.safeParse({ ... })    // returns { success, data?, error? }
await TunnelConfig.fromFile("./cft.yaml")
TunnelConfig.fromYaml("yaml string")
```
