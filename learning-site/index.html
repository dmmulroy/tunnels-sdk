repository map

There are two products sharing one lifecycle idea.

The SDK owns Cloudflare API orchestration and cloudflared process handling. The CLI is a thin command layer that adapts those SDK services into terminal commands and output formats.

SDK package

Published as tunnels. Exposes async/await wrappers plus an advanced Effect API.

packages/tunnels
Public API
src/index.ts exports TunnelClient, expose, domain types, and config parsing.
Effect core
src/effect/services/* contains API, ingress, DNS, routes, vnets, binary, and process services.
Composition
src/effect/layers/Live.ts wires real services; Test.ts provides overridable stubs.

CLI package

Published as tunnels-cli with a tunnels binary built on Effect CLI.

packages/cli
Command tree
src/main.ts registers global flags and top-level commands like expose, create, list, and config.
Service ports
src/services.ts defines CLI-facing interfaces independent of the SDK implementation.
Adapters
src/live-layer.ts maps SDK errors and services to CLI errors, output, and in-memory running tunnel handles.

main flows

Trace these paths first.

Each flow is a thread through the architecture. Follow the files in order, and stop at the tests next to each service.

quick expose

Anonymous URL for a local port

  1. Public call: packages/tunnels/src/wrapper.ts exports async expose(port).
  2. Runtime: it creates a ManagedRuntime from CloudflaredBinary.layer and a manually controlled Scope.
  3. Effect: src/effect/expose.ts ensures the binary, spawns cloudflared tunnel --url http://localhost:<port>, and reads stderr.
  4. Cleanup: closing the scope kills the process; the wrapper exposes close() and Symbol.asyncDispose.
named lifecycle

Create, configure, DNS, routes

1. API create

TunnelOperations.create posts to /accounts/:id/cfd_tunnel with a generated secret.

2. Ingress

IngressManager.set preserves current config and auto-appends http_status:404.

3. DNS

DnsManager.ensure finds the zone and upserts a CNAME to tunnelId.cfargotunnel.com.

4. Routes

RouteManager.add adds private network routes and resolves vnet names when provided.

process runtime

Run and observe a named tunnel

TunnelProcessService.run builds cloudflared tunnel --no-autoupdate run --token ..., attaches finalizers, parses JSON stderr, and maintains status/connectors in refs.

parse stderr into logs + events

Effect mental model

Services are ports. Layers are wiring. Effects are programs.

The code avoids hidden singletons. Production wiring lives in layers, and tests override service ports with fake implementations.

ServiceMap.Service

Declares an interface such as CloudflareApi, DnsManager, or TunnelProcessService.

Layer.effect

Builds a concrete implementation from dependencies. LiveLayer composes them into one runtime.

Scope

Owns tunnel process lifetime. Finalizers kill cloudflared when the scope closes.

Tagged errors

SDK errors are typed. CLI adapters map them to exit codes in packages/cli/src/errors.ts.

watch list

A few details to notice while reading.

  • Package docs describe some aspirational APIs; current code exposes TunnelInfo plus sub-clients rather than a rich Tunnel object.

  • ConfigService and AuthService in the CLI live layer are currently stubs, even though command shells exist.

  • The CLI ingress adapter chooses the first healthy tunnel because its CLI service interface lacks explicit tunnel context.

  • The SDK quick expose wrapper keeps a manually controlled scope; use close() or await using to clean up.
